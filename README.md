=========================================
Docker & Git 開発プロトコル
=========================================

この手順書は、コードの変更からコンテナの起動、
そしてGitHubへの保存までの一連のサイクルをまとめたものです。

-----------------------------------------
【フェーズ1】 開発と確認（Dockerの操作）
-----------------------------------------
コードや設定を変更した後、手元で動かして確認する手順です。

1. コンテナの構築と起動
   コマンド: docker compose up -d --build
   説明: 変更を反映しつつ、バックグラウンドで起動します。

2. 起動状況の確認
   コマンド: docker ps
   説明: コンテナが正しく動いているか、ポート番号を確認します。

3. ログの確認（必要に応じて）
   コマンド: docker compose logs -f
   説明: エラーが起きていないかリアルタイムで監視します。（終了は Ctrl + C）

4. ブラウザで確認
   URL: http://localhost:8080
   説明: 実際の画面を見て動作チェックをします。

5. コンテナの停止と削除
   コマンド: docker compose down
   説明: 作業が終わったら、このコマンドで綺麗にお片付けします。


-----------------------------------------
【フェーズ2】 履歴の保存（Gitの操作）
-----------------------------------------
手元での動作確認が完了したら、その変更を歴史として記録します。

1. 変更状況の確認
   コマンド: git status
   説明: どのファイルが変更・追加されたかを確認します。

2. 記録の準備（ステージング）
   コマンド: git add .
   説明: 現在のディレクトリにあるすべての変更を、次のセーブ対象にします。

3. 歴史に記録（コミット）
   コマンド: git commit -m "変更内容をここに書く"
   説明: 「何をどう変えたか」をメッセージとして残し、セーブします。

4. 履歴の確認（必要に応じて）
   コマンド: git log --oneline
   説明: 今作ったコミットが正しく一番上に追加されているか確認します。


-----------------------------------------
【フェーズ3】 クラウドへの共有（GitHubへPush）
-----------------------------------------
手元のパソコンに記録した歴史を、GitHubへアップロードします。

1. GitHubへ送信
   コマンド: git push
   説明: 手元のコミットをクラウドへ反映させます。

2. 認証（パスワード入力）
   Username: shuaokis-lang
   Password: 発行済みの Personal Access Token (ghp_... から始まる文字列)


-----------------------------------------
【🆘 トラブルシューティング】
-----------------------------------------
「ポートがすでに使われている」などのエラーが出た場合の対処法です。

1. 原因の特定
   コマンド: docker ps
   説明: 現在動いている不要なコンテナの名前やIDを見つけます。

2. 強制終了と削除
   コマンド: docker rm -f <コンテナ名またはID>
   説明: 邪魔なコンテナを強制的に消去して、ポートを解放します。
